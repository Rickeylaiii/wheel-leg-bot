# 轮腿机器人测试上位机

这是一个基于 **Python + PySide6 + pyqtgraph** 的串口测试上位机，用于配合 `wheel-leg-bot.ino` 的 0~11 测试菜单，对机器人硬件进行带图形界面的联机验证。

## 功能

- 串口连接 / 断开
- 一键发送测试命令 `0~11` 和 `h`
- 支持单电机手动命令：`M1F`、`M1R`、`M1S`、`M2F`、`M2R`、`M2S`、`MS`
- 提供“方向确认”面板，可对 `M1/M2` 正反转分别标记“正确/错误”，并自动汇总 `PASS/FAIL`
- 实时解析固件串口输出
- 实时波形显示：
  - 电池电压
  - IMU 姿态角
  - IMU 陀螺仪 / 加速度
  - 电机速度 / 位置
  - 编码器角度
- 原始日志窗口
- CSV 录制导出

## 已适配的固件输出

当前解析器已适配以下串口输出格式：

- `Battery Raw: ... | Filtered: ...`
- `Angle(XYZ): ... | Gyro(XYZ): ... | Acc(XYZ): ...`
- `Encoder1 Angle: ... Vel: ... | Encoder2 Angle: ... Vel: ...`
- `Torque(V) Target=... | M1 Angle: ... Vel: ... | M2 Angle: ... Vel: ...`
- `FOC Result -> M1:... M2:... | Ready:YES/NO`
- `Angle Target: ... | M1: ... | M2: ...`
- `Velocity Target: ... | M1: ... | M2: ...`
- `Stage:... | M1 Target:... Vel:... | M2 Target:... Vel:...`
- `OpenLoop Stage:... | M1 Cmd:... | M2 Cmd:...`
- `Manual OpenLoop | M1 Cmd:... | M2 Cmd:...`
- `Manual Command Applied OpenLoop | M1 Cmd:... | M2 Cmd:...`

## 安装

建议使用 Python 3.10+。

### 1. 安装依赖

在 `host_pc` 目录执行：

```bash
pip install -r requirements.txt
```

### 2. 启动

```bash
python main.py
```

Windows 下也可以直接双击：

```bash
run_gui.bat
```

## 使用流程

1. 烧录并启动 `wheel-leg-bot.ino`
1. 用 USB 将 ESP32 连接到电脑
1. 打开上位机，点击 **刷新串口**
1. 选择正确串口，波特率使用 `115200`
1. 点击 **连接串口**
1. 点击左侧测试按钮。

可用测试项如下：

- `1` 电池
- `2` IMU
- `3` 舵机
- `4` 编码器
- `5` FOC基础
- `6` FOC校准
- `7` 位置闭环
- `8` 速度闭环
- `9` 力矩扫描 / 单电机测试
- `10` 无编码器开环转动测试
- `11` 手动开环模式（配合单电机按钮或自定义命令）

1. 如需保存测试数据，点击 **开始录制 CSV**

手动控制命令如下：

- `M1F`：1号电机正转
- `M1R`：1号电机反转
- `M1S`：1号电机停止
- `M2F`：2号电机正转
- `M2R`：2号电机反转
- `M2S`：2号电机停止
- `MS`：两台电机全部停止

方向确认建议流程：

1. 点击 `11` 进入手动开环模式。
1. 在“单电机手动控制 / 方向确认”面板中点击每一项的“执行”。
1. 观察实际转向后，在对应下拉框中选择“正确”或“错误”。
1. 查看面板底部的自动汇总结果（`PASS / FAIL / 进行中`）。

## 注意事项

- `6/7/8/9` 涉及电机动作，测试前请抬起轮子或做好安全固定。
- `10` 是无编码器开环测试，适合在 AS5600 尚未安装时先验证电机/驱动板是否能转动。
- `11` 和上述手动命令同样工作在无编码器开环模式下，适合单独确认左右电机的正反转方向。
- 当前舵机库只有写入接口，没有回读接口，因此 GUI 不能显示舵机实际到位反馈。
- 当前“力矩测试”是 **电压力矩模式**，不是电流闭环力矩控制。
